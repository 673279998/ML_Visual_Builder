工作流构建

工作流概念

工作流是平台的核心功能，通过拖拽模块构建完整的机器学习流程。每个工作流包含多个节点，节点之间通过连线表示数据流向。

1

拖拽模块

从左侧模块面板拖拽需要的模块到画布中央区域

2

连接模块

从一个模块的输出端口（右侧小圆点）拖拽到另一个模块的输入端口（左侧小圆点）

3

配置模块参数

在右侧面板点击配置按钮打开配置面板，设置相关参数（数据集选择、算法选择、超参数等）

4

执行工作流

点击右上角"执行"按钮运行整个工作流，查看训练结果

详细操作指南

1. 模块拖拽操作

  • 在左侧模块面板找到需要的模块（如"数据集选择"、"分类算法选择"等）
  • 按住鼠标左键，将模块拖拽到画布中央的空白区域
  • 释放鼠标左键，模块会固定在画布上
  • 可以拖拽模块调整位置，右键点击模块可以删除

2. 模块连接操作

  • 每个模块都有输入端口（左侧）和输出端口（右侧）
  • 将鼠标移动到输出端口（小圆点），光标会变成十字形
  • 按住鼠标左键，拖拽到目标模块的输入端口
  • 连接线会自动生成，表示数据流向
  • 可以点击连接线删除连接，或重新连接其他端口

注意：数据流向必须正确，通常从数据输入模块 → 预处理模块 → 算法模块 → 输出模块。

3. 模块配置操作

  • 在右侧面板点击配置按钮打开配置面板
  • 不同模块有不同的配置选项：
    • 数据集选择模块：选择已上传的数据集
    • 算法选择模块：选择具体算法（如随机森林、逻辑回归等）
    • 超参数模块：配置算法超参数（见下文详细说明）
    • 训练控制模块：设置交叉验证、评分指标等
  • 配置完成后点击"确定"保存设置

4. 超参数设置

在算法模块的配置面板中，可以设置以下超参数：

  • 基本超参数：如树模型的max_depth、n_estimators等
  • 交叉验证设置：CV折数（默认5折）、评分指标选择
  • 随机种子：确保结果可复现
  • 训练测试划分：训练集比例（默认0.8）

具体算法的超参数说明见算法指南章节。

典型工作流示例

一个完整的分类任务工作流通常包含以下模块：

  1. 数据集选择模块：选择已配置好的数据集
  2. 预处理模块（可选）：数据清洗、特征编码
  3. 分类算法选择模块：选择具体分类算法（如随机森林）
  4. 超参数调优模块（可选）：配置网格搜索参数
  5. 模型训练模块：执行训练过程

将这些模块按顺序连接起来，配置好每个模块的参数，然后点击"执行"按钮即可开始训练。

数据管理

数据上传

平台支持多种数据格式：

  • CSV文件：逗号分隔值文件，最常用的数据格式
  • Excel文件：支持.xlsx和.xls格式
  • JSON文件：JavaScript对象表示法

上传步骤：

  1. 进入"数据管理"页面
  2. 点击"上传新数据"按钮
  3. 选择文件并设置数据集名称
  4. 点击"上传"按钮

注意：上传成功后，数据会显示在左侧数据集列表中，可以在表格中直接查看和编辑。

列属性配置（关键步骤）

数据上传后，必须对每一列的属性进行配置，这是模型训练的前提：

步骤1：选择列

在数据表格中，点击列名（列标题），右侧会显示该列的属性配置面板。

步骤2：配置数据类型

在右侧属性面板中，选择列的数据类型：

  • 数值型：整数、浮点数等数值数据
  • 文本型：字符串、分类文本数据
  • 日期型：日期时间数据
  • 布尔型：True/False或0/1数据

步骤3：设置目标列

对于有监督学习任务，需要指定目标列（预测目标）：

  • 勾选"是否为目标列"选项
  • 一个数据集只能有一个目标列
  • 目标列的数据类型应与任务类型匹配（分类任务为类别，回归任务为数值）

步骤4：选择编码方式

对于文本型或类别型数据，需要选择合适的编码方式：

  • 标签编码：将类别映射为整数（0,1,2...）
  • 独热编码：为每个类别创建二值特征列
  • 序数编码：按指定顺序为类别分配整数
  • 目标编码：使用目标变量的均值编码
  • 频率编码：使用类别出现频率编码
  • 二进制编码：将整数转换为二进制表示
  • 哈希编码：使用哈希函数映射到固定桶

详细编码方式说明见算法指南章节。

重要：必须为每一列都配置好属性，特别是目标列和编码方式，否则工作流执行会失败。

数据编辑

模型训练

详细训练流程

模型训练是平台的核心功能，通过工作流系统实现完整的训练流程：

步骤1：准备工作流

  • 进入"工作流"页面，创建新的工作流或加载已有工作流
  • 从左侧模块面板拖拽"数据集选择"模块到画布
  • 在右侧面板点击数据集模块的配置按钮，选择已配置好的数据集
  • 确保数据集已正确配置列属性（数据类型、目标列、编码方式）

步骤2：选择算法模块

  • 根据任务类型拖拽相应的算法模块：
    • 分类任务：拖拽"分类算法选择"模块
    • 回归任务：拖拽"回归算法选择"模块
    • 聚类任务：拖拽"聚类算法选择"模块
    • 降维任务：拖拽"降维算法选择"模块
  • 连接数据集模块的输出端口到算法模块的输入端口
  • 在右侧面板点击算法模块的配置按钮，选择具体算法（如随机森林、逻辑回归等）

步骤3：配置训练参数

  • 超参数设置：在算法模块配置面板中设置算法特定参数
    • 树模型：max_depth、n_estimators、min_samples_split等
    • 线性模型：C、penalty、solver等
    • 神经网络：hidden_layer_sizes、learning_rate、max_iter等
  • 交叉验证设置：设置CV折数（默认5折）、评分指标
  • 数据划分：训练集比例（默认0.8）、随机种子
  • 训练控制：最大迭代次数、早停设置、并行计算等

步骤4：添加超参数调优（可选）

  • 拖拽"超参数调优"模块到画布
  • 连接算法模块到超参数调优模块
  • 在右侧面板点击超参数调优模块的配置按钮，选择调优方法：
    • 网格搜索：遍历所有参数组合
    • 随机搜索：随机采样参数空间
    • 贝叶斯优化：智能搜索最优参数
  • 设置调优参数：参数网格、迭代次数、CV折数等

步骤5：执行训练

  • 点击右上角"执行"按钮开始训练
  • 查看实时训练进度和日志
  • 监控CPU、内存使用情况
  • 训练过程中可以暂停、停止或查看中间结果

步骤6：查看训练结果

  • 训练完成后自动跳转到结果页面
  • 查看评估指标和可视化图表
  • 分析特征重要性（如果算法支持）
  • 保存模型到模型库

模型结果查看与管理

训练完成后，可以在"模型管理"页面查看和管理所有训练的模型：

1. 模型列表查看

  • 进入"模型管理"页面查看所有已训练的模型
  • 每个模型卡片显示：算法名称、训练时间、主要评估指标
  • 支持按算法类型、训练时间、性能指标排序
  • 支持搜索和筛选模型

2. 模型详情查看

  • 点击模型卡片或"查看详情"按钮进入模型详情页
  • 查看完整的训练结果：
    • 评估指标：准确率、F1分数、AUC-ROC等
    • 可视化图表：学习曲线、混淆矩阵、特征重要性等
    • 超参数信息：训练使用的具体参数值
    • 训练日志：详细的训练过程记录
  • 支持图表交互：缩放、平移、下载图表

3. 模型删除与管理

  • 单个删除：点击模型卡片的"删除"按钮删除单个模型
  • 批量删除：勾选多个模型，点击"批量删除"按钮
    • 勾选模型前的复选框选择模型
    • 点击"全选"选择所有模型
    • 点击"批量删除"按钮删除选中的模型
  • 清理垃圾文件：点击"清理垃圾文件"按钮删除未使用的模型文件、临时文件等
  • 模型导出：导出模型为joblib格式文件，可在其他环境中使用

4. 多余文件清理

平台会自动清理以下类型的多余文件：

  • 未使用的处理后数据集：训练完成后生成的中间数据文件
  • 临时模型文件：训练过程中生成的临时文件
  • 旧的预测结果：历史预测结果文件
  • 缓存文件：系统缓存文件

清理操作不会删除正在使用的文件和数据库记录。

模型输出结果说明

模型训练完成后，平台会生成以下类型的输出结果：

1. 评估指标结果

  • 分类模型：准确率、精确率、召回率、F1分数、AUC-ROC、混淆矩阵
  • 回归模型：均方误差(MSE)、平均绝对误差(MAE)、R²分数、解释方差
  • 聚类模型：轮廓系数、Calinski-Harabasz指数、Davies-Bouldin指数
  • 降维模型：解释方差比、累计解释方差

2. 可视化结果

  • 学习曲线：训练集和验证集的学习过程
  • 特征重要性：各特征对预测结果的重要性排序
  • 混淆矩阵热图：分类模型的预测结果分布
  • ROC曲线：分类模型的性能曲线
  • 残差图：回归模型的预测误差分布
  • 聚类可视化：聚类结果的二维/三维可视化
  • 降维可视化：高维数据在低维空间的投影

3. 模型文件

  • 模型文件：训练好的模型保存为joblib格式
  • 预处理组件：数据预处理步骤的序列化文件
  • 特征编码器：特征编码器的序列化文件
  • 配置文件：模型配置和超参数信息

4. 训练报告

  • 训练摘要：算法类型、训练时间、数据规模等
  • 参数设置：使用的超参数和训练参数
  • 性能分析：模型在不同数据集上的表现
  • 错误分析：模型的主要错误类型和改进建议

模型预测

详细预测流程

对于有监督学习任务（分类和回归），可以在"模型预测"页面使用训练好的模型进行预测：

步骤1：选择预测模型

  • 进入"模型预测"页面
  • 在"选择模型"下拉菜单中选择已训练的模型
  • 系统会自动显示模型信息：
    • 算法类型：分类、回归等
    • 算法名称：随机森林、逻辑回归等
    • 训练时间：模型训练的时间
    • 输入特征：模型需要的特征列信息
  • 查看数据预处理详情：数据集格式、特征列信息、预处理方法等

步骤2：准备预测数据

平台支持两种输入方式：

方法一：手动输入（推荐）
  • 选择"手动输入"模式
  • 在文本框中输入JSON格式的数据
  • 数据格式要求： 
    [
  {"feature1": 5.1, "feature2": 3.5, "feature3": 1.4},
  {"feature1": 6.2, "feature2": 2.8, "feature3": 4.8}
]
  • 注意事项：
    • 特征名称必须与训练时完全一致
    • 数值类型必须正确（整数、浮点数）
    • 分类特征需要与训练时相同的编码
    • 支持单条或多条数据同时预测
方法二：选择数据集
  • 选择"选择数据集"模式
  • 从下拉菜单中选择已上传的数据集
  • 系统会自动使用该数据集的所有数据进行预测
  • 适用场景：批量预测、测试集评估

步骤3：配置预测参数

  • 返回概率（仅分类模型）：勾选"返回预测概率"选项
    • 对于分类模型，可以返回每个类别的预测概率
    • 概率值范围0-1，所有类别概率之和为1
    • 可用于不确定性分析、阈值调整等
  • 输出格式：选择预测结果的输出格式
    • 简洁格式：只返回预测值
    • 详细格式：返回预测值、概率、置信度等
    • 可视化格式：生成可视化图表
  • 批量处理：设置批量预测的批次大小
    • 大数据集可以分批次预测，避免内存溢出
    • 默认批次大小：1000条/批次

步骤4：执行预测

  • 点击"开始预测"按钮开始预测
  • 查看实时预测进度和状态
  • 预测过程中可以：
    • 暂停：暂停当前预测任务
    • 停止：停止预测并清理临时文件
    • 查看日志：查看详细的预测过程日志
  • 预测完成后自动显示结果

步骤5：查看预测结果

预测完成后，平台会显示以下结果：

1. 表格结果
  • 预测值表格：每条数据的预测结果
  • 概率表格（如果启用）：每个类别的预测概率
  • 置信度：预测结果的置信度分数
  • 支持表格排序、筛选、搜索功能
2. 可视化分析
  • 预测分布图：预测结果的分布情况
  • 概率分布图：预测概率的分布情况
  • 错误分析图：预测错误的样本分析
  • 特征贡献图：各特征对预测结果的贡献度
3. 统计摘要
  • 预测统计：预测值的统计信息（均值、标准差等）
  • 类别分布（分类）：各预测类别的数量和比例
  • 性能评估（如果有真实标签）：准确率、误差等指标

预测数据输入详细说明

预测数据的输入格式和注意事项：

1. JSON格式要求

手动输入时必须使用正确的JSON格式：

// 单条数据预测
{"feature1": 5.1, "feature2": 3.5, "feature3": 1.4}

// 多条数据预测（推荐）
[
  {"feature1": 5.1, "feature2": 3.5, "feature3": 1.4},
  {"feature1": 6.2, "feature2": 2.8, "feature3": 4.8},
  {"feature1": 5.8, "feature2": 2.7, "feature3": 5.1}
]

常见错误：

  • 缺少逗号分隔符
  • 字符串未加引号
  • 数组格式不正确
  • 特征名称拼写错误

2. 特征匹配要求

  • 特征名称：必须与训练时完全一致（包括大小写）
  • 特征顺序：不需要与训练时相同，系统会自动匹配
  • 特征数量：必须包含所有训练时使用的特征
  • 缺失特征：不允许有缺失特征，必须提供所有特征值

提示：在选择模型后，系统会显示模型需要的特征列表，请参考该列表准备数据。

3. 数据类型要求

  • 数值特征：整数或浮点数，如5、3.14、-2.5
  • 分类特征：字符串或整数，需要与训练时相同的编码
    • 如果训练时使用标签编码，预测时使用相同的整数编码
    • 如果训练时使用独热编码，预测时使用原始字符串
  • 布尔特征：true/false或0/1
  • 日期特征：ISO格式字符串，如"2023-01-15"

4. 数据预处理

预测数据会自动应用与训练时相同的预处理：

  • 特征缩放：标准化、归一化等
  • 特征编码：标签编码、独热编码等
  • 缺失值处理：均值填充、中位数填充等
  • 特征选择：只使用训练时选择的特征子集

注意：预处理参数（如均值和标准差）使用训练时计算的值，确保预测的一致性。

特征编码方式详解

平台支持7种特征编码方式，适用于不同的数据场景：

1. 标签编码 (Label Encoding)

原理: 将每个类别映射为一个唯一的整数（0, 1, 2...）。

特点: 简单高效，不增加特征维度，但会引入类别间的顺序关系。

适用场景: 树模型（决策树、随机森林、XGBoost等）通常能处理这种顺序关系，类别数量较少时效果较好。

示例: ["北京", "上海", "广州"] → [0, 1, 2]

注意事项: 线性模型可能将整数编码误解为有序关系，使用时需注意。

2. 独热编码 (One-Hot Encoding)

原理: 为每个类别创建一个新的二值特征列（0或1）。

特点: 完全消除类别间的顺序关系，但会增加特征维度（维度爆炸问题）。

适用场景: 需要消除顺序关系的场景，如线性模型（逻辑回归、SVM）、神经网络等，类别数量较少（建议少于15个）时效果最佳。

示例: ["北京", "上海", "广州"] → 北京:[1,0,0], 上海:[0,1,0], 广州:[0,0,1]

注意事项: 类别过多时会导致维度灾难，需要配合特征选择或降维。

3. 序数编码 (Ordinal Encoding)

原理: 按照指定的顺序为类别分配整数，保持类别间的顺序关系。

特点: 保留类别顺序信息，适合有自然顺序的类别。

适用场景: 教育程度（小学、中学、大学）、产品等级（低、中、高）等有序类别，树模型和线性模型都能利用这种顺序信息。

示例: ["小学", "中学", "大学"] → [0, 1, 2]（保持顺序）

注意事项: 需要预先定义类别顺序，否则可能引入错误的顺序关系。

4. 目标编码 (Target Encoding)

原理: 使用目标变量的均值（或其他统计量）来编码每个类别。

特点: 编码值包含目标信息，能提高模型性能，但容易导致目标泄漏。

适用场景: 高基数类别特征（如城市、产品ID），树模型通常能更好地利用这种编码。

示例: 城市"北京"的目标均值是500，则所有"北京"编码为500

注意事项: 必须在交叉验证中谨慎使用，防止数据泄漏。

5. 频率编码 (Frequency Encoding)

原理: 使用类别在数据集中出现的频率（或比例）进行编码。

特点: 不增加特征维度，编码值反映类别的重要性。

适用场景: 高基数类别，树模型和线性模型都能使用，树模型通常能更好地利用频率信息。

示例: 类别"北京"出现100次，总样本1000，则编码为0.1（频率）或100（计数）

注意事项: 稀有类别可能被赋予相似的频率值，失去区分度。

6. 二进制编码 (Binary Encoding)

原理: 先将类别映射为整数，再将整数转换为二进制表示，每位二进制作为一个特征。

特点: 维度介于标签编码和独热编码之间，平衡了效率和效果。

适用场景: 类别数量中等（几十到几百个），需要控制特征维度，各种模型都适用。

示例: 8个类别需要3位二进制（2³=8），如类别3编码为011

注意事项: 二进制位之间没有实际意义，可能影响模型解释性。

7. 哈希编码 (Hash Encoding)

原理: 使用哈希函数将类别映射到固定数量的特征桶中。

特点: 特征维度固定，适合类别数量极多（如用户ID、IP地址）。

适用场景: 高基数类别（成千上万个），在线学习系统，各种模型都适用。

示例: 使用8位哈希，将任意类别映射到256个特征桶之一

注意事项: 存在哈希冲突，不可逆，不适合需要解释性的场景。

分类算法详解

平台支持11种分类算法，每种算法都有特定的超参数配置。在工作流中在右侧面板点击算法模块的配置按钮即可打开配置面板设置这些参数。

1. 逻辑回归 (Logistic Regression)

原理: 使用逻辑函数将线性回归的输出映射到[0,1]区间，用于二分类或多分类任务。

特点: 可解释性强，计算效率高，适合线性可分数据。

适用场景: 二分类问题，需要概率输出的场景。

可配置超参数:

参数名 类型 默认值 范围/选项 说明
C 浮点数 1.0 0.001 - 100 正则化强度的倒数，值越小正则化越强
penalty 选择 l2 l1, l2, elasticnet, none 正则化类型，l1产生稀疏解，l2防止过拟合
max_iter 整数 100 50 - 1000 最大迭代次数，优化算法收敛的最大迭代
solver 选择 lbfgs lbfgs, liblinear, newton-cg, sag, saga 优化算法，不同算法支持不同的正则化类型

模型输出结果: 分类概率、预测类别、系数权重、特征重要性（线性系数）

可视化结果: ROC曲线、混淆矩阵、特征系数条形图、决策边界图

2. 随机森林分类器 (Random Forest Classifier)

原理: 集成学习方法，通过构建多个决策树并投票决定最终分类结果。

特点: 抗过拟合能力强，可处理高维数据，提供特征重要性评估。

适用场景: 各种分类任务，特别是特征维度较高的数据集。

可配置超参数:

参数名 类型 默认值 范围/选项 说明
n_estimators 整数 100 10 - 500 决策树的数量，数量越多模型越稳定但训练越慢
max_depth 整数 None 1 - 50 树的最大深度，None表示不限制，控制模型复杂度
min_samples_split 整数 2 2 - 20 节点分裂所需的最小样本数，防止过拟合

模型输出结果: 预测类别、分类概率、特征重要性、每棵树的预测结果

可视化结果: 特征重要性条形图、决策树可视化、混淆矩阵、ROC曲线

3. 决策树分类器 (Decision Tree Classifier)

原理: 基于树结构的分类模型，通过特征划分构建决策路径。

特点: 可解释性强，可视化效果好，无需特征缩放。

适用场景: 需要可解释性的分类任务，特征重要性分析。

可配置超参数:

参数名 类型 默认值 范围/选项 说明
max_depth 整数 None 1 - 50 树的最大深度，None表示不限制
min_samples_split 整数 2 2 - 20 节点分裂所需的最小样本数
min_samples_leaf 整数 1 1 - 20 叶子节点所需的最小样本数

模型输出结果: 预测类别、决策树结构、特征重要性、叶子节点信息

可视化结果: 决策树可视化图、特征重要性条形图、决策路径图、混淆矩阵

4. 梯度提升分类器 (Gradient Boosting Classifier)

原理: 集成学习方法，通过串行训练弱学习器，每轮纠正前一轮的错误。

特点: 预测精度高，可处理复杂非线性关系。

适用场景: 对预测精度要求高的分类任务。

可配置超参数:

参数名 类型 默认值 范围/选项 说明
n_estimators 整数 100 10 - 500 弱学习器（树）的数量
learning_rate 浮点数 0.1 0.01 - 1.0 学习率，控制每棵树的贡献权重
max_depth 整数 3 1 - 20 树的最大深度，控制模型复杂度

模型输出结果: 预测类别、分类概率、特征重要性、每棵树的贡献

可视化结果: 特征重要性图、学习曲线、决策路径、SHAP值分析

5. XGBoost分类器 (XGBoost Classifier)

原理: 优化的梯度提升算法，加入正则化项防止过拟合。

特点: 速度快，精度高，支持并行计算。

适用场景: 大规模数据集，竞赛级分类任务。

可配置超参数:

参数名 类型 默认值 范围/选项 说明
n_estimators 整数 100 10 - 500 弱学习器（树）的数量
learning_rate 浮点数 0.1 0.01 - 1.0 学习率，控制每棵树的贡献权重
max_depth 整数 6 1 - 20 树的最大深度，控制模型复杂度

模型输出结果: 预测类别、分类概率、特征重要性、每棵树的贡献

可视化结果: 特征重要性图、学习曲线、SHAP值分析、决策路径

6. LightGBM分类器 (LightGBM Classifier)

原理: 基于直方图的梯度提升算法，使用叶子生长策略。

特点: 内存占用小，训练速度快，适合大规模数据。

适用场景: 大规模高维数据集，需要快速训练的场景。

可配置超参数:

参数名 类型 默认值 范围/选项 说明

num_leaves 整数 31 10 - 100 叶子数量，控制模型复杂度

模型输出结果: 预测类别、分类概率、特征重要性、叶子权重

7. CatBoost分类器 (CatBoost Classifier)

原理: 梯度提升算法，专门优化了类别特征处理。

特点: 自动处理类别特征，抗过拟合能力强。

适用场景: 包含大量类别特征的数据集。

可配置超参数:

参数名 类型 默认值 范围/选项 说明
iterations 整数 100 10 - 500 迭代次数（树的数量）
learning_rate 浮点数 0.1 0.01 - 1.0 学习率，控制每棵树的贡献权重
depth 整数 6 1 - 16 树的最大深度，控制模型复杂度

可视化结果: 特征重要性图、学习曲线、类别特征编码效果、决策路径

8. SVM分类器 (Support Vector Machine Classifier)

原理: 寻找最优超平面，最大化类别间隔。

特点: 在高维空间表现好，适合小样本数据。

适用场景: 文本分类、图像识别等小样本高维数据。

可配置超参数:

参数名 类型 默认值 范围/选项 说明

kernel 选择 rbf linear, poly, rbf, sigmoid 核函数类型，决定特征空间的映射方式

模型输出结果: 预测类别、分类概率（如果支持）、支持向量、决策函数值

10. KNN分类器 (K-Nearest Neighbors Classifier)

原理: 基于距离度量，根据最近邻样本的类别进行预测。

特点: 简单直观，无需训练过程，适合小数据集。

适用场景: 样本量较小的分类任务。

可配置超参数:

参数名 类型 默认值 范围/选项 说明

  • 需要概率输出: 逻辑回归、朴素贝叶斯、高斯混合模型

超参数调优

超参数调优概述

超参数调优是机器学习中的重要步骤，通过搜索最优的超参数组合来提高模型性能。平台支持三种调优方法：网格搜索、随机搜索和贝叶斯优化。

使用步骤：

  1. 在工作流中添加"超参数调优"模块
  2. 连接算法模块到超参数调优模块
  3. 在右侧面板点击超参数调优模块的配置按钮打开配置面板
  4. 选择调优方法并设置参数
  5. 执行工作流开始调优

1. 网格搜索 (Grid Search)

原理：遍历所有可能的参数组合，寻找最优解。

特点：

  • 优点：能找到全局最优解（在给定参数网格内）
  • 缺点：计算成本高，参数组合数量呈指数增长
  • 适用场景：参数空间较小，需要精确最优解

用户输入格式：在超参数调优节点中，您需要为每个参数指定一个搜索范围（列表）。平台会自动遍历所有可能的组合。

示例：

{
  "max_depth": [3, 5, 7, 10],
  "n_estimators": [50, 100, 200],
  "learning_rate": [0.01, 0.1, 0.3]
}

使用说明：

  1. 参考算法指南中每个算法的参数范围建议
  2. 为每个参数指定一个列表，包含要搜索的具体值
  3. 参数组合数量 = 各参数列表长度的乘积
  4. 建议从粗粒度搜索开始，逐步细化

注意事项：参数组合过多会导致计算时间急剧增加，请合理设置搜索范围。

2. 随机搜索 (Random Search)

原理：在参数空间中随机采样，寻找较优解。

特点：

  • 优点：计算效率高，适合高维参数空间
  • 缺点：可能错过全局最优解
  • 适用场景：参数空间较大，需要快速找到较好解

用户输入格式：在超参数调优节点中，您需要为每个参数指定一个搜索分布或范围。平台会从这些分布中随机采样参数组合。

示例：

{
  "max_depth": [3, 5, 7, 10],
  "n_estimators": [50, 100, 200],
  "learning_rate": [0.01, 0.1, 0.3]
}

或者使用scipy.stats分布：

{
  "max_depth": "randint(3, 10)",
  "learning_rate": "uniform(0.01, 0.3)",
  "n_estimators": "randint(50, 300)"
}

使用说明：

  1. 参考算法指南中每个算法的参数范围建议
  2. 为每个参数指定一个列表（离散值）或分布（连续范围）
  3. 设置n_iter参数控制随机采样次数（默认50次）
  4. 随机搜索比网格搜索更适合高维参数空间

注意事项：随机搜索不保证找到全局最优解，但通常能在较短时间内找到较好的解。

3. 贝叶斯优化 (Bayesian Optimization)

原理：基于贝叶斯定理，利用先前评估结果指导后续搜索。

特点：

  • 优点：智能搜索，效率最高
  • 缺点：实现复杂，需要更多计算资源
  • 适用场景：计算成本高的模型，需要高效搜索

用户输入格式：在超参数调优节点中，您需要为每个参数指定一个搜索空间（范围）。平台会使用贝叶斯优化算法智能搜索最优参数。

示例：

{
  "max_depth": (3, 10),
  "learning_rate": (0.01, 0.3),
  "n_estimators": (50, 300)
}

对于离散参数：

{
  "kernel": ["linear", "rbf", "poly"],
  "C": (0.1, 10.0),
  "gamma": (0.01, 1.0)
}

使用说明：

  1. 参考算法指南中每个算法的参数范围建议
  2. 为连续参数指定范围（元组形式，如(最小值, 最大值)）
  3. 为离散参数指定列表（如["选项1", "选项2"]）
  4. 设置n_iter参数控制迭代次数（默认30次）
  5. 贝叶斯优化适合计算成本高的模型，能更高效地找到最优解

注意事项：贝叶斯优化需要scikit-optimize库支持，如果未安装会自动回退到随机搜索。

交叉验证设置

所有超参数调优方法都使用交叉验证来评估参数性能：

CV折数选择

  • 5折交叉验证：默认设置，平衡计算成本和稳定性
  • 10折交叉验证：更稳定但计算成本更高
  • 3折交叉验证：快速但可能不稳定，适合大数据集

评分指标

  • 分类任务：accuracy, f1, roc_auc, precision, recall
  • 回归任务：neg_mean_squared_error, r2, neg_mean_absolute_error
  • 聚类任务：silhouette_score, calinski_harabasz_score

随机种子

设置random_state确保结果可复现，相同的随机种子会产生相同的结果。

调优结果解读

调优完成后，平台会提供以下结果：

  • 最佳参数：找到的最优超参数组合
  • 最佳分数：最佳参数在验证集上的得分
  • 参数重要性：各参数对模型性能的影响程度
  • 学习曲线：参数变化对性能的影响趋势
  • 并行坐标图：多维参数空间的性能可视化

建议：调优完成后，使用最佳参数重新训练模型，并在独立测试集上评估性能。

常见问题

Q: 数据上传后为什么无法在工作流中使用？

A: 数据上传后必须配置列属性才能在工作流中使用。请按以下步骤操作：

  1. 进入"数据管理"页面
  2. 点击数据表格中的列名（列标题）
  3. 在右侧属性面板中配置：
    • 数据类型：数值型、文本型、日期型等
    • 目标列：对于有监督学习，勾选"是否为目标列"
    • 编码方式：对于文本/类别数据，选择合适的编码方式
  4. 为每一列都配置好属性后，数据就可以在工作流中使用了

Q: 工作流执行失败，提示"数据集未配置"怎么办？

A: 这通常是因为数据集没有正确配置列属性。请检查：

  • 是否为目标列设置了正确的数据类型（分类任务为类别，回归任务为数值）
  • 是否为文本/类别列选择了合适的编码方式
  • 是否所有必需的列都已配置属性
  • 可以在数据管理页面重新检查每列的配置

Q: 超参数怎么设置？有哪些注意事项？

A: 超参数设置在工作流的算法模块中：

  • 设置位置：在右侧面板点击算法模块的配置按钮打开配置面板
  • 参数类型：
    • 算法特定参数（如树模型的max_depth、n_estimators）
    • 训练控制参数（如max_iter、random_state）
    • 交叉验证设置（CV折数、评分指标）
  • 注意事项：
    • 参数值必须在合理范围内（参考算法指南中的建议）
    • 复杂的参数组合可以使用超参数调优模块自动搜索
    • 设置随机种子确保结果可复现

Q: 超参数调优怎么设置？交叉验证怎么设置？

A: 使用"超参数调优"模块：

  1. 在工作流中添加"超参数调优"模块
  2. 连接算法模块到超参数调优模块
  3. 在右侧面板点击超参数调优模块的配置按钮选择调优方法：
    • 网格搜索：遍历所有参数组合
    • 随机搜索：随机采样参数空间
    • 贝叶斯优化：智能搜索最优参数
  4. 设置交叉验证：
    • CV折数：通常5或10折
    • 评分指标：分类用accuracy，回归用neg_mean_squared_error
    • 随机种子：确保结果可复现
  5. 执行工作流开始调优

Q: 模型结果怎么查看？有哪些可视化图表？

A: 训练完成后可以在"模型管理"页面查看结果：

  • 评估指标：准确率、F1分数、AUC-ROC、均方误差等
  • 可视化图表：
    • 学习曲线：训练集和验证集的学习过程
    • 特征重要性：各特征对预测结果的重要性
    • 混淆矩阵：分类模型的预测结果分布
    • ROC曲线：分类模型的性能曲线
    • 残差图：回归模型的预测误差分布
  • 超参数信息：训练使用的具体参数值
  • 训练日志：详细的训练过程记录

Q: 模型怎么删除？模型相关的东西怎么删除？

A: 在"模型管理"页面进行删除操作：

  • 单个删除：点击模型卡片的"删除"按钮
  • 批量删除：
    1. 勾选要删除的模型（点击模型前的复选框）
    2. 点击"批量删除"按钮
    3. 确认删除操作
  • 清理垃圾文件：点击"清理垃圾文件"按钮删除未使用的文件

注意：删除操作会同时删除模型文件、预处理组件、预测结果等相关文件。

Q: 怎么执行预测？预测数据怎么输入？

A: 在"模型预测"页面执行预测：

  1. 选择已训练的模型
  2. 准备预测数据：
    • 手动输入：输入JSON格式数据，如[{"feature1": 5.1, "feature2": 3.5}]
    • 选择数据集：从已上传的数据集中选择
  3. 配置预测参数（如是否返回概率）
  4. 点击"开始预测"按钮
  5. 查看预测结果和可视化分析

数据格式要求：特征名称必须与训练时一致，数据类型必须正确。

Q: 怎么保存工作流？怎么加载工作流？

A: 工作流保存和加载功能：

  • 保存工作流：
    1. 在工作流页面右上角点击"保存工作流"
    2. 输入工作流名称和描述
    3. 点击"保存"按钮
  • 加载工作流：
    1. 在工作流页面右上角点击"加载工作流"
    2. 从列表中选择已保存的工作流

  • 管理工作流：在"工作流管理"页面可以重命名、删除、导出、导入工作流

Q: 正确的操作流程是什么？

A: 推荐的标准操作流程：

  1. 数据管理：上传数据 → 点击列名配置属性（数据类型、目标列、编码方式）
  2. 工作流构建：拖拽模块 → 连接模块 → 在右侧面板点击配置按钮设置参数
  3. 模型训练：执行工作流 → 查看训练结果
  4. 模型管理：查看模型详情 → 管理模型（删除、导出等）

  • 分类算法（11种）：逻辑回归、随机森林、决策树、梯度提升、XGBoost、LightGBM、CatBoost、SVM、朴素贝叶斯、KNN、MLP
  • 回归算法（11种）：线性回归、岭回归、Lasso、弹性网络、决策树回归、随机森林回归、梯度提升回归、XGBoost回归、LightGBM回归、SVR、MLP回归
  • 聚类算法（5种）：K-Means、层次聚类、DBSCAN、高斯混合模型、谱聚类

Q: 编码方式有哪些？怎么选择？

A: 平台支持7种编码方式：

  • 标签编码：将类别映射为整数，适合树模型
  • 独热编码：为每个类别创建二值列，适合线性模型
  • 序数编码：按指定顺序编码，适合有序类别
  • 目标编码：使用目标均值编码，适合高基数类别
  • 频率编码：使用类别频率编码
  • 二进制编码：将整数转换为二进制表示
  • 哈希编码：使用哈希函数映射

详细说明请参考算法指南章节的编码方式详解。

Q: 模型输出的结果和可视化结果怎么解读？

A: 模型输出包括：

  • 评估指标：量化模型性能，如准确率越高越好，均方误差越低越好
  • 学习曲线：训练集和验证集误差随训练进程的变化，用于诊断过拟合/欠拟合
  • 特征重要性：显示各特征对预测的贡献度，用于特征选择
  • 混淆矩阵：显示分类模型的预测结果分布，对角线上的值越多越好
  • ROC曲线：曲线下面积(AUC)越大，模型性能越好
  • 残差图：残差应随机分布，无明显模式

详细解读方法请参考模型训练章节的结果说明。
